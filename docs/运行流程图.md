## 环境差异对比

| 项目 | Cursor 环境 | VS Code 环境 |
|------|------------|-------------|
| API 密钥 | 不需要 | 必须配置 |
| 规则来源 | `.cursorrules` 文件 | `.cursorrules` 文件 |
| 调用方式 | Cursor 内置命令 | HTTP API 调用 |
| 结果填充 | 自动填充到 Git 面板 | 复制到剪贴板 |

## 关键服务说明

### 1. CursorRulesService

负责 `.cursorrules` 文件的管理：
- `exists()` - 检测文件是否存在
- `read()` - 读取文件内容
- `generate()` - 生成默认模板
- `promptToGenerate()` - 提示用户生成文件

### 2. PromptService

负责生成 AI 提示词：
- 读取 `.cursorrules` 内容
- 添加代码变更（diff）
- 生成完整的提示词

### 3. AIService

负责调用 AI 服务：
- Cursor 环境：调用 `cursor.generateGitCommitMessage` 命令
- VS Code 环境：调用 OpenAI/Anthropic/自定义 API

## 配置项说明

| 配置项 | Cursor 环境 | VS Code 环境 |
|--------|------------|-------------|
| `apiProvider` | 忽略（默认 cursor） | 必填 |
| `apiKey` | 不需要 | 必填 |
| `customPrompt` | 忽略 | 可选（高级覆盖） |
| 其他 API 配置 | 忽略 | 可选 |

## 注意事项

1. **Cursor 内置功能**：在 Cursor 环境中，插件通过 `cursor.generateGitCommitMessage` 命令触发 Cursor 的内置 AI 功能，Cursor 会自动读取项目的 `.cursorrules` 文件。

2. **统一规则配置**：无论是 Cursor 还是 VS Code 环境，都使用项目根目录的 `.cursorrules` 文件来定义提交规则，确保团队成员使用统一的规则。

3. **自动提示生成**：如果项目中没有 `.cursorrules` 文件，首次使用时插件会提示用户生成一个默认的规则文件。
