# AI Auto Commit 插件开发指南

## 快速开始

### 1. 安装依赖
```bash
npm install
```

### 2. 编译代码
```bash
npm run compile
```

### 3. 调试插件
1. 按 `F5` 启动调试
2. 在新打开的窗口中测试插件功能

### 4. 打包插件
```bash
npm install -g @vscode/vsce
vsce package
```

## 项目结构

```
ai-auto-commit/
├── src/
│   ├── extension.ts           # 插件入口文件
│   └── services/
│       ├── gitService.ts      # Git 服务
│       ├── aiService.ts       # AI 服务
│       ├── promptService.ts   # 提示词服务
│       └── commitService.ts   # 提交服务
├── out/                       # 编译输出目录
├── package.json               # 插件配置
├── tsconfig.json              # TypeScript 配置
└── README.md                  # 说明文档
```

## 核心模块说明

### 1. GitService
负责与 Git 仓库交互：
- `isGitRepository()`: 检查是否是 Git 仓库
- `getStagedDiff()`: 获取已暂存的代码变更
- `getChangedFiles()`: 获取变更的文件列表
- `getCurrentBranch()`: 获取当前分支名称

### 2. AIService
负责调用 AI API：
- 支持 OpenAI、Anthropic、自定义 API
- 统一的接口设计，易于扩展新的 AI 服务
- 完善的错误处理

### 3. PromptService
负责生成提示词：
- 支持默认模板和自定义模板
- 支持多种提交格式（Conventional Commits、简单格式）
- 可以整合上下文信息（分支、文件列表等）

### 4. CommitService
负责处理提交操作：
- 提交信息格式化
- 格式验证
- Git 提交执行

## 扩展开发

### 添加新的 AI 服务提供商

在 `AIService` 类中添加新方法：

```typescript
private async callCustomProvider(prompt: string, config: AIServiceConfig): Promise<string | null> {
  // 实现自定义 AI 服务的调用逻辑
  const response = await this.axiosInstance.post(
    'https://your-api-endpoint.com/v1/chat',
    {
      // API 请求参数
    },
    {
      headers: {
        'Authorization': `Bearer ${config.apiKey}`,
      },
    }
  );
  
  return response.data.message?.trim() || null;
}
```

然后在 `generateCommitMessage` 方法中添加新的 case。

### 添加新的提交格式

在 `PromptService` 中添加新的提示词模板，并在 `CommitService` 中添加对应的验证逻辑。

### 添加新的配置项

1. 在 `package.json` 的 `contributes.configuration.properties` 中添加配置项
2. 在代码中使用 `vscode.workspace.getConfiguration('aiCommit').get('yourConfigKey')` 获取配置

## 测试建议

1. **单元测试**：为每个 Service 编写单元测试
2. **集成测试**：测试完整的提交流程
3. **错误测试**：测试各种错误情况（无 API 密钥、无 Git 仓库等）

## 发布流程

1. 更新版本号（package.json）
2. 更新 CHANGELOG.md
3. 运行测试
4. 打包：`vsce package`
5. 发布：`vsce publish`
