# AI Auto Commit 插件功能说明

## 核心功能对比

### sy-ai-commit 插件功能
1. ✅ AI 生成提交信息
2. ✅ 多种 AI 模型支持
3. ✅ 自定义提示词
4. ✅ 项目级别设置
5. ✅ API 密钥配置
6. ✅ 规范提交记录

### 本插件实现的功能
1. ✅ **AI 生成提交信息** - 自动分析代码变更，生成规范的提交信息
2. ✅ **多种 AI 模型支持** - 支持 OpenAI、Anthropic、自定义 API
3. ✅ **自定义提示词** - 支持自定义提示词模板
4. ✅ **多种提交格式** - 支持 Conventional Commits、简单格式
5. ✅ **安全配置** - 支持 API 密钥配置
6. ✅ **智能格式化** - 自动清理和格式化提交信息
7. ✅ **用户确认** - 生成后允许用户查看、编辑再提交

## 使用流程

1. **修改代码** → 在工作区中修改代码文件
2. **暂存文件** → 使用 `git add` 暂存要提交的文件
3. **触发命令** → 按 `Ctrl+Shift+P`，输入 "生成 AI 提交信息"
4. **查看结果** → 查看生成的提交信息
5. **确认提交** → 选择"使用此提交信息"或"编辑后提交"

## 配置说明

### 基础配置
- `aiCommit.apiProvider`: 选择 AI 服务提供商（openai/anthropic/custom）
- `aiCommit.apiKey`: 输入你的 API 密钥
- `aiCommit.model`: 选择或输入模型名称（如 gpt-3.5-turbo, claude-3-opus 等）

### 高级配置
- `aiCommit.customPrompt`: 自定义提示词模板，使用 `{diff}` 作为占位符
- `aiCommit.commitType`: 提交格式类型（conventional/simple/custom）
- `aiCommit.maxTokens`: 最大生成 token 数（默认 200）
- `aiCommit.temperature`: AI 生成温度（0-2，默认 0.7）

### 示例配置

#### OpenAI 配置
```json
{
  "aiCommit.apiProvider": "openai",
  "aiCommit.apiKey": "sk-...",
  "aiCommit.model": "gpt-3.5-turbo"
}
```

#### Anthropic 配置
```json
{
  "aiCommit.apiProvider": "anthropic",
  "aiCommit.apiKey": "sk-ant-...",
  "aiCommit.model": "claude-3-opus-20240229"
}
```

#### 自定义提示词示例
```
请根据以下代码变更生成提交信息，要求：
1. 使用中文
2. 格式：类型(范围): 简短描述
3. 详细描述代码变更的内容和原因

代码变更：
{diff}
```

## 技术实现

### 架构设计
```
extension.ts (入口)
├── GitService (Git 操作)
│   ├── 检查 Git 仓库
│   ├── 获取代码变更 (diff)
│   └── 获取变更文件列表
├── PromptService (提示词生成)
│   ├── 默认提示词模板
│   ├── 自定义提示词处理
│   └── 上下文信息整合
├── AIService (AI 调用)
│   ├── OpenAI API
│   ├── Anthropic API
│   └── 自定义 API
└── CommitService (提交处理)
    ├── 提交信息格式化
    ├── 格式验证
    └── Git 提交执行
```

### 关键特性
1. **异步处理** - 使用 VS Code Progress API 显示进度
2. **错误处理** - 完善的错误捕获和提示
3. **安全性** - API 密钥仅存储在 VS Code 配置中
4. **可扩展性** - 支持自定义 AI 提供商和提示词
