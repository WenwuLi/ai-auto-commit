# AI Auto Commit 调试指南

## 📋 调试前准备

### 1. 安装依赖（已完成 ✅）
```bash
npm install
```

### 2. 编译项目（已完成 ✅）
```bash
npm run compile
```

## 🚀 三种调试方式

### 方式一：快速调试（推荐）

**最适合日常开发调试**

#### 步骤：
1. **启动监听模式**（在终端运行，保持后台运行）
```bash
npm run watch
```

2. **按 F5 启动调试**
   - VS Code 会自动打开一个新窗口
   - 这个新窗口是 "扩展开发宿主窗口"
   - 你的插件会在这个窗口中运行

3. **测试插件功能**
   - 在新窗口中按 `Ctrl+Shift+P` 打开命令面板
   - 输入并运行 `生成 AI 提交信息` 命令
   - 观察原窗口中的调试输出

#### 优点：
- ✅ 代码修改后自动重新编译
- ✅ 不需要手动重启调试
- ✅ 开发体验最佳

#### 配置详情：
```12:14:.vscode/launch.json
      "name": "运行扩展",
      "type": "extensionHost",
      "request": "launch",
```

### 方式二：断点调试

**最适合定位问题、追踪代码执行流程**

#### 步骤：
1. **在源代码中设置断点**
   - 打开 `src/extension.ts` 或任意服务文件
   - 在需要调试的行号左侧点击，会出现红色圆点

2. **启动调试**
   - 按 `F5` 启动调试
   - 代码执行到断点处会自动暂停

3. **使用调试面板**
   - **变量**：查看当前作用域的变量值
   - **监视**：添加表达式监视
   - **调用堆栈**：查看函数调用链
   - **断点**：管理所有断点

4. **控制执行**
   - `F5` 继续执行
   - `F10` 单步跳过
   - `F11` 单步进入
   - `Shift+F11` 单步跳出

#### 示例断点位置：
```28:35:src/extension.ts
export function activate(context: vscode.ExtensionContext) {
  console.log('AI Auto Commit 插件已激活');

  // 初始化服务
  const gitService = new GitService();
  const aiService = new AIService();
  const promptService = new PromptService();
  const commitService = new CommitService();
```

#### 调试输出位置：
- 打开 **调试控制台**（Debug Console）
- 查看扩展宿主（Extension Host）的输出
- `console.log()` 会在这里显示

### 方式三：完整重新编译调试

**最适合需要完整重新构建的场景**

#### 步骤：
1. **停止之前的调试进程**

2. **手动编译**
```bash
npm run compile
```

3. **按 F5 启动调试**

#### 优点：
- ✅ 确保使用最新编译的代码
- ✅ 适合修复编译错误后测试

### 4. 常见问题排查

#### 问题：断点不生效
**解决方案**：
- 检查 `tsconfig.json` 中 `sourceMap: true`
- 确认使用的是编译后的 `.js.map` 文件
- 重新运行 `npm run compile`

#### 问题：调试窗口没有响应
**解决方案**：
- 关闭调试窗口
- 停止所有调试进程
- 重新运行 `npm run watch`
- 再次按 F5

#### 问题：看不到日志输出
**解决方案**：
- 打开 **输出面板**（Ctrl+Shift+U）
- 在下拉菜单选择 **扩展宿主（Extension Host）**
- 检查 `console.log` 是否正确调用

#### 问题：代码修改后没反应
**解决方案**：
- 确认 `npm run watch` 正在运行
- 检查终端是否有编译错误
- 在新窗口中重新运行命令

## 📝 调试配置说明

### launch.json 配置详解

```1:15:.vscode/launch.json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "运行扩展",
      "type": "extensionHost",
      "request": "launch",
      "args": [
        "--extensionDevelopmentPath=${workspaceFolder}"
      ],
      "outFiles": [
        "${workspaceFolder}/out/**/*.js"
      ],
      "preLaunchTask": "${defaultBuildTask}"
    },
```

- **type: "extensionHost"**：VS Code 扩展调试类型
- **args**：传递参数给扩展宿主窗口
- **outFiles**：调试时的 JavaScript 文件路径
- **preLaunchTask**：启动前自动执行的任务

### tasks.json 配置详解

```1:16:.vscode/tasks.json
{
  "version": "2.0.0",
  "tasks": [
    {
      "type": "npm",
      "script": "watch",
      "problemMatcher": "$tsc-watch",
      "isBackground": true,
      "presentation": {
        "reveal": "never"
      },
      "group": {
        "kind": "build",
        "isDefault": true
      }
    },
```

- **isBackground**：后台任务，不阻塞
- **problemMatcher**：匹配编译错误并显示
- **isDefault**：F5 时自动运行此任务

## 🎯 实际调试流程示例

### 场景：调试 AI API 调用失败

1. **设置断点**
   - 在 `src/services/aiService.ts` 的 API 调用处设置断点

2. **启动调试**
   - 按 F5，打开新窗口

3. **触发调试**
   - 在新窗口中运行 `生成 AI 提交信息` 命令

4. **检查变量**
   - 查看 `config.apiKey` 是否正确
   - 查看 `requestData` 格式是否正确
   - 查看 `response` 内容

5. **查看错误**
   - 在调试控制台查看完整错误信息
   - 检查网络请求是否成功

6. **修复问题**
   - 在原窗口修改代码
   - 等待 watch 自动编译
   - 在新窗口重新测试
